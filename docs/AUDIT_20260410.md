## 2. Findings and fixes

### 2.1 Python - src / tests

**core/models.py - Pydantic v2 deprecation**
The model used the v1-style inner `class Config` with `use_enum_values = True`.
Pydantic v2 emits a PydanticDeprecatedSince20 warning and will remove this in
v3. Replaced with `model_config = ConfigDict(use_enum_values=True)` and added
`ConfigDict` to the import line. Behavior is unchanged; the warning is gone.

**tests/unit/test_search.py - failing fallback test**
The test created an empty `search.db` but not `events.jsonl`. The production
`search.command()` has an intentional guard that only runs the linear fallback
when both files exist, so the fallback branch was never exercised and the test
failed. Fix: also write an empty `events.jsonl` stub in the test setup so the
branch under test is actually reached. The production safety gate was left
intact - the test now matches its own docstring.
